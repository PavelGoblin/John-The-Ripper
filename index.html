What is a Password Hash?

User Password → Hash Algorithm → Stored Hash
Example:
Password: hello123
MD5: 482c811da5d5b4bc...
  • 🔒 One-way function
  • 📏 Fixed-length output
  • ✅ Designed for verification
Login Process: User enters password → System hashes it → Compares with stored hash
4

Main Goal of the Attack

🎯

Objective

Recover the original plaintext password from the stolen hash

Since hashes cannot be mathematically reversed, attackers guess passwords repeatedly until the generated hash matches the stolen one.

Why Dictionary Attacks Work

Easy passwords

Rule-Based Attack

Applies transformation rules to dictionary words to simulate human habits.

// Rule transformations
hello → Hello123 # Capitalize + year
admin → admin@2025 # Symbol + year
football → Football# # Capitalize + symbol
💡 Efficiency: Multiplies wordlist size exponentially without increasing storage requirements.

Rainbow Table Attack

Precomputed hash lookup technique

📊 Precomputed hashes stored in massive lookup databases
⚡
Instant hash-to-password mapping without recomputation
🚫 Modern Defense: Salting and strong algorithms (bcrypt/Argon2) make rainbow tables nearly useless today
23

What is Salting?

password + random_salt → unique_hash
🛡️
Prevents Reuse
Identical passwords produce different hashes
🚫
Blocks Rainbow Tables
Precomputed tables become invalid
💡 Best Practice: Every password should use a unique, cryptographically secure salt
24

Weak vs Strong Hashing

❌ Weak (Vulnerable)

  • 🔴 MD5
  • 🔴 SHA-1
  • 🔴 LM Hash (Legacy Windows)

✅ Strong (Secure)

  • 🟢 bcrypt (Adaptive cost)
  • 🟢 Argon2 (Memory-hard)
  • 🟢 PBKDF2 (Iteration-heavy)
25

Real-World Attack Example

  1. 🌐 Website database breached → user hashes leaked
  2. 🧩 Attackers identify hash type (e.g., SHA-256)
  3. 📖 Load targeted wordlists (rockyou, leaked creds)
  4. ⚙️ Apply rule-based transformations
  5. 💥 Run John the Ripper → weak passwords recovered
  6. 🔄 Credential reuse attempted on other platforms

Password Entropy & MFA

📊
Password Entropy
Measures unpredictability. Higher entropy = exponentially harder to crack. Longer passwords dramatically increase security.
🔐
Multi-Factor Auth
Even if password is cracked, additional verification stops access. SMS codes, authenticator apps, hardware tokens.
🛡️ Defense in Depth: Strong passwords + MFA = Nearly impenetrable account security
29

Modern Defenses & Ethics

🛡️ Modern Defenses

  • ✅ Strong passwords & MFA
  • ✅ bcrypt/Argon2 hashing
  • ✅ Account lockout & rate limiting
  • ✅ Continuous monitoring

⚖️ Ethical vs Illegal

  • 🟢 Authorized: Legal, educational, defensive
  • 🔴 Unauthorized: Illegal, unethical, criminal
